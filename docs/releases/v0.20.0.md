title: Review-first workflows, explainable MCP, and safer automation
description: RepoPilot 0.20 turns local repository analysis into a complete change-review workflow for developers, agents, and GitHub pull requests.
---

## Highlights

- Scan and review now lead with `PASS`, `REVIEW`, or `BLOCK`, provide progressive detail, and share canonical decisions, verification plans, and dependency impact paths across output formats.
- GitHub Action reviews compare base and head deterministically, focus sticky comments on new and changed findings, report resolved findings, and upload stable JSON, delta, SARIF, and Markdown artifacts.
- MCP adds workspace-bound analysis handles, pagination, bounded responses, review-signal explanations, retained-analysis discovery, and stored-only context when safe finding replay is unavailable.
- Guided help and `repopilot init` provide review-first workflows plus deterministic GitHub Action and generic or client-specific MCP bootstrap files without restoring legacy commands.
- Analysis is faster and safer through shared parse facts, incremental graph/cache reuse, bounded parallelism, recoverable cache writes, root-confined agent paths, centralized redaction, determinism tests, and cross-platform smoke coverage.

Recommended workflows:

Review local work:

```bash
repopilot review .
repopilot review . --detail full
```

Review a branch before merge:

```bash
repopilot review . --base origin/main
```

Mark the starting point before an agent or developer works:

```bash
repopilot snapshot
```

After the work is complete, review everything since that marker:

```bash
repopilot review --since-snapshot
```

Bootstrap integrations:

```bash
repopilot init --github-action
repopilot init --mcp-client generic
```

## Compatibility

The reduced top-level command surface remains unchanged. Removed legacy commands are not restored.

Finding IDs and baseline matching remain stable. Report, AI artifact, Action, and MCP contracts advance additively where documented. Existing CLI gates and exit-code semantics remain unchanged.

## Upgrade

```bash
cargo install repopilot --version 0.20.0 --force
npm install -g repopilot@0.20.0
```
